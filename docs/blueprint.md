# **App Name**: PDF Unlocker Pro

## Core Features:

- Multi-file Upload & Management: Allow users to drag-and-drop or click to upload multiple PDF files. Only .pdf files are accepted. Displays uploaded files within a dedicated section, showing a file icon, name (truncated with full name on hover), muted file size, a right-aligned status badge, and a remove button that appears subtly on hover. When files are dragged over the upload area, its border darkens slightly, background gets a subtle blue-gray tint, and the upload icon/helper text become more prominent, avoiding dramatic effects. This section is the primary focus upon page load and transitions to focus on the file list after uploads.
- Unified Action Panel: Provides a single password input field for users to enter a common password for all uploaded PDFs, with a show/hide toggle. This panel also contains the main action button. The primary action button is full-width on mobile and right-aligned on desktop, displaying 'Unlock PDFs' by default and 'Unlocking PDFs...' with a subtle inline spinner during processing. A secondary 'Clear All' button is lighter and less visually dominant. The panel disables actions during processing to prevent duplicate submissions, and displays responsive progress feedback and live-feeling processing updates.
- Batch PDF Unlocking (Backend): A backend service receives uploaded PDFs and the provided password. It intelligently attempts to unlock each file; if the password field is empty, it attempts to open PDFs without a password as a valid action. It identifies and categorizes outcomes: successful unlocking, incorrect password, corrupted files, or already unlocked PDFs. Unlocked files are renamed 'unlocked_originalfilename.pdf' in memory.
- Streamlined Status & Progress Display: Presents a calm and easy-to-scan UI with clear per-file status updates during processing. Simplified statuses include: Ready, Processing, Unlocked, Already Unlocked, Wrong Password, and Failed. An overall progress bar and contextual text (e.g., 'Processing X of Y files') provide a non-overwhelming sense of progress. Processing rows should have minimal, calm visual activity.
- Automated ZIP Packaging & Download: After processing, all successfully unlocked and 'already unlocked' PDF files are bundled into a single ZIP archive. This archive is automatically downloaded to the user, with a format 'unlocked_pdfs_YYYYMMDD_HHmm.zip'. The summary and individual file statuses remain visible after download.
- Post-processing Results Summary: After processing and ZIP download, a clean, reassuring summary card appears, detailing counts for total files, successfully unlocked, already unlocked, wrong password, and failed files. This card also includes options to 'Download completed successfully', 'Retry failed files', and 'Clear all and start over', providing a clear overview and next steps.
- Product-Friendly Error Handling: Implements comprehensive error handling with polished, user-friendly messages. Specific messages include: 'No files uploaded.', 'No unlockable PDFs were found.' (if none could be unlocked), and 'Some files could not be processed.' (for partial failures). Backend issues, such as 'Couldn’t connect to the server. Please try again.', are also handled gracefully. Errors like an empty password field for encrypted PDFs are handled as part of the unlocking attempt, not as a validation error.
- Exported Document Branding: Provides an optional branding feature for exported PDFs, summary pages, or printable reports. A toggle 'Include OROMedu logo in exported files' is enabled by default, allowing users to disable it. The logo 'OROMedu' (OROM in soft pastel blue, edu in soft pastel red, clean sans-serif) is small, tasteful, and positioned in the top-left corner of the exported document. The logo area is visually clean, never overlapping existing content, and reserves 16-24px padding around the logo.
- Export Options: Before exporting, users can choose options such as 'Include OROMedu logo', 'Include export date', 'Include summary statistics', and 'Include detailed per-file results'. These options are enabled by default and presented in a calm, minimal settings area prior to export.

## Style Guidelines:

- Primary Palette: A sophisticated, muted gray (#F8F9FA) for the main background to convey professionalism and calm. Main cards are pristine white (#FFFFFF) providing a clean canvas. For drag-and-drop state, the background becomes a subtle blue-gray tint.
- Accent Color: A restrained, deep charcoal gray or a very subtle, desaturated blue (#343A40 or similar) for key interactive elements, borders, and typography, offering a strong but not overwhelming contrast.
- Status Colors: Use a subtle, sophisticated palette for status badges: 'Ready' → muted gray, 'Processing' → blue-gray, 'Unlocked' → soft green, 'Already Unlocked' → neutral slate, 'Wrong Password' → soft amber, and 'Failed' → muted red. Avoid bright, playful hues.
- Branding Colors (OROMedu): 'OROM' in soft pastel blue (#7BA7FF) and 'edu' in soft pastel red (#FF8E8E) for the optional logo in exported documents.
- Primary Font: 'Inter' (sans-serif) for all textual elements. Emphasize its clean lines and high legibility, leveraging different weights and sizes to establish a clear visual hierarchy rather than relying on decorative effects.
- Branding Font (OROMedu): Clean sans-serif appearance for the optional 'OROMedu' logo, medium or semi-bold weight, with slight letter spacing for a refined look. Small size relative to the page, not dominant.
- Utilize a minimal, clean, line-based icon set. Icons should be functional and subtle, supporting content without drawing undue attention, e.g., a simple file icon for list items, a lock/unlock icon for status, and a subtle inline spinner for processing.
- Overall Structure: A focused, centered container with a restrained two-card layout for most states. The primary card combines the upload area and file list. The secondary card groups the password input, action buttons, and progress feedback. A separate results summary card appears only after processing.
- Spacing & Visual Hierarchy: Employ generous, consistent spacing and a strong grid system to create a calm, organized, and easily scannable interface. Typography, white space, and subtle border treatments are the primary tools for establishing hierarchy.
- Elements & Shapes: Use crisp, well-defined shapes with restrained but modern rounding: Cards: 12px, Buttons: 10px, Inputs: 10px. Status badges should be pill-shaped. Avoid heavy shadows; instead, use extremely subtle shadows with soft blur and low opacity, preferring thin borders over strong shadows to define card elevation.
- Button Layout: The primary action button is full-width on mobile, right-aligned on desktop. The secondary 'Clear All' button is lighter, less visually dominant, and styled as an outline button.
- Exported Document Layout: For exported reports or printable summary pages, use a clean white background with generous margins. A dedicated top header area includes the optional OROMedu logo on the left and an optional export date on the right, followed by a thin divider line. Main content starts below the divider with generous spacing. Avoid placing content too close to the top edge. The document should feel professional, clean, well-spaced, easy to scan, with clear section headings, consistent spacing, muted divider lines, subtle table styling if needed, comfortable margins, and strong typography hierarchy. The 'OROMedu' logo is positioned in the top-left corner, small and tasteful, with 16-24px padding, ensuring it never overlaps existing text or content.
- Subtle Transitions: Incorporate smooth, understated hover effects for interactive elements (e.g., remove button in file list, main action button) and gentle transitions for status changes or the appearance of new sections (like the results summary card), ensuring a fluid but non-distracting user experience. Processing rows should have minimal, calm visual activity. Drag-and-drop state changes should be subtle, avoiding dramatic scaling, glow effects, or overly animated behavior.
